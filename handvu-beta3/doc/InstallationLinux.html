Installation on Linux

There is currently no binary distribution for Linux or cygwin, so you have to compile HandVu on your system. That's much better, anyway.

Requirements

  • Linux (tested with Redhat 9.0 on 2.4.20-8 kernel and on Fedora2 on 2.6.9-1.11 and 2.6.10 kernels, all with dc1394 firewire interface),
  • a 1.5 GHz or faster CPU (performance increases superlinearly with a faster CPU),
  • OpenCV,
  • a camera with 320x240 or greater resolution; either firewire, USB, or other frame grabber, that works with the OpenCV demos or with ARToolKit.
  • only for the ARToolKit demo: the ARToolKit 2.61, OpenGL, GLUT, and SDL.

Source package

  • Get all the required packages and install them in their default locations.
  • Unpack the archive into a directory of your choice and cd into it. Type
    ./configure
    make
    (make install doesn't do much at the moment)
  • Options to the configure process are:
    --without-artk do not attempt to build hv_ARtk demo [build it]
    --without-opencv do not even use OpenCV for file i/o [use OpenCV]
    --with-magick use ImageMagick=abspath for file i/o [use OpenCV]
    --with-type=type IntegralImage types: [float], uint

Troubleshooting and Tips

  • ARToolKit might not compile due to libdc1394 compatibility issues. This patch, courtesy of Steve DiVerdi, might help the situation: video.c.patch, to be applied to ARToolKit2.61/lib/SRC/VideoLinux1394Cam/video.c with the command "patch < video.c.patch".
  • Note that paths for ./configure --with must be absolute (no ~tilde).
